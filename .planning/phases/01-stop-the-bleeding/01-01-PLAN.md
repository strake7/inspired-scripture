```javascript
export function stripHtml(str) {
  return str?.replace(/<\/?[^>]+(>|$)/g, '')
}

export function truncate(str, maxLength) {
  if (!str || str.length <= maxLength) return str
  const trimmed = str.slice(0, maxLength)
  const lastSpace = trimmed.lastIndexOf(' ')
  return (lastSpace > 0 ? trimmed.slice(0, lastSpace) : trimmed) + '...'
}
```

Two changes to `lib/studies.js`:

**Fix 1 — Title trimming (per D-02, META-01, META-04):**
On line 49, change:
```javascript
study.title = stripHtml(fLine)
```
to:
```javascript
study.title = stripHtml(fLine).trim()
```
This adds `.trim()` after `stripHtml()` to remove the trailing `\n` that comes from the first line of each study HTML file. The first line is extracted via `fullContent.slice(0, fullContent.indexOf('\n') + 1)` which includes the newline character. `stripHtml` strips HTML tags but does not trim whitespace.

This single fix resolves META-01 (og:title, structured data, breadcrumbs) AND META-04 (og:image:alt in meta.jsx line 51 consumes `title` directly).

**Fix 2 — Description extraction (per D-03, META-02):**
In `extractDescription()` (lines 9-29), the regex on line 13-15 matches headers like `Introduction: Some text` and captures group 3 as the description. But some studies have `Introduction.` (with a period, not colon) as the header text — e.g., `Introduction. Job is the oldest book...`. The current regex `/(Introduction|Background|Overview)(: )?(.+)/` captures `: ` optionally, but when the header uses a period+space pattern like `Introduction. `, the period and space end up in capture group 3.

Change the regex on lines 13-15 from:
```javascript
const headerMatch = plainText.match(
  /(Introduction|Background|Overview)(: )?(.+)/,
)
```
to:
```javascript
const headerMatch = plainText.match(
  /(Introduction|Background|Overview)[.:;]?\s*(.+)/,
)
```

And update the capture group reference on line 16 from `headerMatch?.[3]` to `headerMatch?.[2]`:
```javascript
if (headerMatch?.[2]) return truncate(headerMatch[2], MAX_DESCRIPTION_LENGTH)
```

This regex now:
1. Matches the header word (Introduction/Background/Overview)
2. Optionally skips a trailing period, colon, or semicolon plus any whitespace
3. Captures the rest as group 2 (the actual description text)
